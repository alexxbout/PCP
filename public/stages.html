Développeur Web Full Stack 2021-2022 | Okmont
Contexte : Okmont est une société située 2 Rue de la Perrière, à Melesse, spécialisée dans la rénovation énergétique. La société accompagne les clients souhaitant entreprendre des démarches de rénovation énergétique dans leur habitat. Une équipe de techniciens au cœur de la société intervient sur les chantiers. Les clients sont mis en relation avec les commerciaux de l'entreprise qui les accompagnent jusqu'à la fin des démarches et jusqu'à 10 ans après la pose d'un chantier. Afin d'avoir un regard sur les ventes réalisées, vendues ou annulées, les commerciaux utilisent une application web leur permettant de rentrer leurs ventes et prospection, offrant des données pleinement exploitables pour les réunions commerciales tous les lundis.

Cette application a été réalisée par deux étudiants du lycée Victor et Hélène Basch en 2018, et est actuellement reprise par mes soins. La société a rencontré le besoin de corriger certaines anomalies et souhaite ajouter quelques fonctionnalités qui seront présentées plus bas.

Mes principaux interlocuteurs sont le dirigeant de l’entreprise ainsi que les commerciaux, qui sont les premiers concernés par l’application, mais qui sont également moins présents la semaine puisque en déplacement chez les clients. Le lundi est donc pour moi une journée très intéressante puisqu’elle me permet de prendre note des remarques et attentes concernant l’outil.

Technologies utilisées : Les technologies utilisées sont une application web développée en PHP, CSS, HTML, JavaScript avec une base de données MySQL et quelques framework JS (AmCharts, JQuery) ou encore CSS (TailwindCSS).
Présentation du framework TailwindCSS : TailwindCSS est un framework CSS complètement personnalisable, basé sur le principe de classes utilitaires. L’outil est très efficace pour réaliser la partie front d’un site web, mais comme ce dernier est très complet et couvre la plupart des classes css qui existent, le fichier de style devient vite très lourd pour le navigateur, près de 4 megabit ! De ce fait, avec sa version 2.0, TailwindCSS proposait d'exécuter une commande dont le principe était de scanner tous les fichiers où l’on utilisait les classes utilitaires de l'outil, afin de proposer un fichier de style sur mesure et donc beaucoup moins lourd et sans les classes non utilisées. Mais avec la version 3.0, la technologie “Just-in-Time” permet de générer à la demande les styles dont on a besoin. Les modifications sont prises en compte en temps réel sur le fichier de style offrant un gain de temps et de productivité étant donné qu’avec la version précédente, il fallait ré-exécuter la commande (dont le temps d'exécution variait en fonction de la quantité de fichiers) avant chaque déploiement sur le serveur de production.

Un des points forts de ce framework est également la simplicité de concevoir une interface aussi bien adaptée pour ordinateurs que pour mobiles. Nommés les "breakpoints", ils permettent d’appliquer un style ou plusieurs seulement si la taille de l’écran ne dépasse pas celle précisée. Très pratique dans le cas de l’application de la société étant donné que les commerciaux utilisent essentiellement leur smartphone quand ils sont en déplacement.
Architecture client / serveur :
Sur le serveur de production : La base de données MySQL est hébergée sur le même serveur privé virtuel où se trouve les fichiers de l’application. Le choix de l’hébergeur de ce service qui à été fait par la société est OVH. Sur le serveur de développement : Pour développer l’application web, j’utilise mon ordinateur portable personnel sous Windows 11 ainsi que de l’utilitaire XAMPP, ce qui me permet d’avoir un serveur linux avec les services Apache et MySQL ainsi qu’un interpréteur PHP. J’utilise visual studio code pour développer l’application, dont le code source est hébergé sur un repo privé disponible sur mon compte github me permettant de garder une trace de l’avancement du projet.

Description Le stage se déroulera en 3 trois étapes, à savoir :

Une première partie d’une durée de deux semaines sera consacrée aux fonctionnalités d’utilisation et de correction de bugs avec la prise en compte d’une possible reprise du projet par un autre étudiant impliquant la mise en œuvre des bonnes pratiques étudiées en cours.

Etudes de faisabilité L'idée de rendre portable l'application web a été proposée par le dirigeant de l'entreprise. J'ai alors proposé de réaliser un travail sur la partie back de l'application, afin de la rendre plus modulaire ainsi que de rendre son code plus maintenable à l'avenir. Toujours sur le principe de modèle vue et contrôleur, la nouvelle application web sera développé sous le framework CodeIgniter. L'idée est de placer l'application sur un seul serveur web, qui hébergera également la base de données. Le service s'adresse aux sociétés qui réalisent des travaux d'amélioration d'habitat. Les utilisateurs pourront se créer un compte et souscrire à un abonnement. Dès lors, ils deviendront automatiquement administrateurs de leur offre et pourront ajouter des collaborateurs à leur plateforme. Lorsqu'un collaborateur se connectera, il sera automatiquement redirigé sur le portail de son agence.
